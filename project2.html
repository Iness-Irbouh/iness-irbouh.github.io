Projet 2 — Prédiction du Churn Client

Ce projet utilise plusieurs méthodes de Machine Learning pour prédire si un client quitte la banque. Il inclut : nettoyage des données, exploration, modélisation (kNN, Logistique, Naïve Bayes, LDA, QDA), comparaisons de performances et visualisations avancées.

Objectif du projet

L’objectif est de prédire la variable Exited (1 = client sortant) en utilisant un ensemble de variables socio-démographiques et bancaires. Ce projet m’a permis de mettre en œuvre un workflow complet :

  • nettoyage et transformation des données ;
  • exploration statistique ;
  • mise en place de plusieurs modèles de classification ;
  • analyse comparative des performances ;
  • création de graphiques explicatifs.

1. Description des données

Base utilisée : Churn_Modelling.csv (10 000 clients).

Variables principales :

  • Âge
  • Genre
  • Pays
  • Solde bancaire
  • Score crédit
  • Ancienneté
  • Nombre de produits
  • Statut actif

Variable cible :

Exited → 1 si le client quitte la banque, 0 sinon.

Colonnes supprimées car inutiles :

  • RowNumber
  • CustomerId
  • Surname

2. Préparation et exploration

Étapes réalisées :

  • transformation des variables en facteurs ;
  • vérification des valeurs manquantes ;
  • séparation des données : 75% train / 25% test ;
  • visualisations exploratoires (Age vs Balance, distributions par classe).

Ces analyses permettent de comprendre les tendances avant de construire les modèles.

3. Modèles testés

kNN (k-Nearest Neighbors)

Test du modèle avec plusieurs valeurs de k. Visualisation de la frontière de décision pour comprendre la séparation des classes.

Régression Logistique

Deux versions créées :

  • modèle simple ;
  • modèle amélioré via AIC (sélection automatique des meilleures variables).

Analyses effectuées :

  • coefficients et probabilités prédites ;
  • matrice de confusion ;
  • accuracy, sensibilité, spécificité ;
  • courbe ROC et aire AUC.

Naïve Bayes

Modèle rapide et simple, utilisé pour tester la classification probabiliste. Frontière de décision affichée.

LDA & QDA

Méthodes discriminantes permettant de visualiser la séparation linéaire ou quadratique des classes.

Comparaison basée sur :

  • accuracy ;
  • sensibilité ;
  • spécificité ;
  • matrices de confusion.

4. Comparaison finale

Les performances des modèles (Logistique, kNN, Naïve Bayes, LDA, QDA) ont été regroupées et comparées via leurs indicateurs clés :

  • accuracy ;
  • sensibilité ;
  • spécificité ;
  • courbes ROC.

Cette comparaison permet d'identifier le meilleur modèle pour prédire le churn client.

5. État actuel du projet

La partie technique est totalement terminée : modèles, graphiques et analyse. Prochaines étapes :

  • rédiger le mémoire ;
